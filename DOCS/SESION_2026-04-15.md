### 2. Refactor de `sharepoint_sync.py` — Sincronización Incremental Robusta

#### Antes:
- Sincronizaba desde una fecha fija → lento, redundante
- Un solo error detenía todo el proceso
- Sin reportes de errores

#### Después:
- **Ventana incremental:** Solo procesa cambios de las últimas 12 horas
- **Bucle infinito:** Corre cada 1 hora sin intervención manual
- **Manejo de errores individual:** Cada registro se intenta por separado; si falla, se anota y continúa
- **Validación pre-inserción:** Antes de subir una OP, verifica si ya existe en DB por su **número de OP** (no por ID interno)
- **Mapa de OPs recargado** después de procesar órdenes, para que TV/UN/Pagos puedan vincular OPs recién creadas

#### Lógica de duplicados resuelta:
- Si SharePoint manda la OP `7191` que ya existe en Supabase con el mismo número → **se saltea silenciosamente**
- Si `unidades_negocio` tiene un registro duplicado (misma OP + misma Unidad de Negocio) → **se saltea y se reporta**
- El criterio de unicidad es el **número de OP real** (campo `op_numero`), no el ID interno

### 3. Sistema de Reportes Duales (nuevo)

Se eliminó `DOCS/reporte_huerfanos.txt` y se reemplazó por dos archivos separados generados al final de cada ciclo:

#### `DOCS/sincronizacion_exitosa.txt`
- Lista todas las OPs que se actualizaron correctamente
- Agrupadas por tabla: `[ORDEN DE PUBLICIDAD]`, `[TV]`, `[UNIDADES_NEGOCIO]`, `[PAGOS]`
- Formato: `• OP: 7300  (SP_ID: 1234)`

#### `DOCS/sincronizacion_errores.txt`
- Detalla cada registro que falló o se salteó
- Muestra: Lista | OP | SP_ID | Motivo (DUPLICADO / HUÉRFANO / ERROR DB)
- Permite identificar exactamente qué corregir en SharePoint
